# 实现路线图（Roadmap）

> 本文是 LangGenix 的**实现里程碑总纲**：定义每个版本「做什么、依赖什么、做到什么程度算交付」。
> 它**不展开**每个模块的详细任务拆解——那部分按需进行（见下方[「工作流约定」](#工作流约定)）。
>
> 配套阅读：功能纲领 [`features.md`](./features.md)、决策记录 [`decisions.md`](./decisions.md)、术语 [`glossary.md`](./glossary.md)。

---

## 工作流约定

本 roadmap 是**骨架**，不是**逐行施工图**。两层文档分工：

| 层级 | 文件 | 内容 | 何时产出 |
| --- | --- | --- | --- |
| **里程碑总纲** | `docs/roadmap.md`（本文） | 版本目标 / 交付物 / 验收线 / 依赖 | 现在已定，随大决策变更而更新 |
| **模块详细拆解** | `docs/modules/<模块名>.md` | 数据模型 / 接口 / 文件清单 / 子任务 / 边界 | **按需**，逐模块走下方流程 |

**逐模块开发流程**（每次开工一个模块/特性时）：

```
我提出「开始做模块 X」
   → 你先给【实现方案 + 待我拍板的疑问】
   → 我拍板取舍
   → 落成 docs/modules/X.md（详细拆解文档）
   → 再按该文档实现 + 跑门禁 + commit
```

> 即：`docs/modules/` 现在是空的，**不预先写满所有模块细节**——避免在未动手时凭空设计出伪通用接口（呼应 ADR-003「抽象由第二个实例倒逼」）。

## 代码库起点（事实基线）

LangGenix 长在 **NextDevTpl** 这套 Next.js 16 SaaS 模板上。盘点真实起点：

**已就位（直接复用）**
- 认证 `better-auth`、`user` 表 / `session` / `account`（学习档案 0.4 将关联 `user` 表）
- AI 抽象 `src/lib/ai/openai.ts` 的 `chatCompletion()`（OpenAI/DeepSeek/MiMo 可切换）→ v0.2 生成复用
- 异步处理 `inngest`（已装，v0.2 起承载 AI 长任务）
- ORM `drizzle-orm` + `drizzle-kit`（`pnpm db:push` / `db:generate`）
- 图表 `recharts`（v0.5 统计复用）、存储 `storage`（R2/S3，TTS 音频用）
- 配置目录 `src/config/`（`subscription-plan.ts` 模式 → 语言包 0.3 将来放这）
- 三级 server action 客户端 `src/lib/safe-action.ts`（`protectedAction` 提供 `ctx.user`）

**零代码（本路线图要从头建）**
- `src/db/schema.ts` 中**无任何** card / deck / review / article / quiz 表
- `ts-fsrs` **未安装**（ADR-004 指定库，v0.1 第一件事之一）
- `src/features/` 下**无**语言学习模块（现有 15 个均为模板商业模块）

**保留不动**（ADR：自用优先，不为 KPI 妥协，但不删模板资产）
`credits` / `payment` / `subscription` / `pseo` / `marketing` / `blog` / `analytics` 等模板商业模块**原样保留**、不在本路线图范围内；未来若需分享/变现可复用。语言学习功能新建于独立 feature 模块，不与其纠缠。

## 里程碑总览

每个版本都遵循 ADR 的核心信条「**小步可用**」——做完即可立刻自用，而非「做完一切才能用」。

| 版本 | 主题 | 一句话目标 | 覆盖模块 | 核心依赖 |
| --- | --- | --- | --- | --- |
| **v0.1** | Anki 底座 | 单人能真正用它背一轮词 | 0.1 / 0.2 / 0.4 / 1.x（核心） | 无（地基） |
| **v0.2** | 语境化生成 ⭐ | 用「最近学的词」生成短文并学习 | 2B.1 / 2B.3 / 0.5(雏形) / Inngest | v0.1 的卡片数据 |
| **v0.3** | 语法文章 + 记忆增强 | 语法点驱动文章 + 一键助记/易混辨析 | 2B.2 / 2A.x | v0.2 的生成管线 |
| **v0.4** | 造卡回流 + 测验 | AI 造卡沉淀 + 测验检验闭环 | 2C.x / 3.x | v0.2/v0.3 有内容可沉淀 |
| **v0.5** | 统计与成就 | 复习/时间/成就统计，制造成就感 | 4.x / 0.5(完整) | 前序数据攒够 |

**顺序逻辑**（与 `features.md` 模块依赖完全一致）：
> 先有数据底座（v0.1）→ 才有东西可生成（v0.2/v0.3）→ 才有东西可沉淀检验（v0.4）→ 数据攒够才做统计（v0.5）。

## v0.1 · Anki 底座

> **杀手锏的地基**。模块 0（卡片模型 + SRS 引擎）是一切的根，必须先定对（ADR：地基先行）。

**目标**：打通「单人能真正用它背一轮词」的最小闭环。

**交付物**
- 卡片 / 牌组数据模型（模块 0.1）——平台核心数据底座
- SRS 排期引擎（模块 0.2）——默认 **FSRS**（`ts-fsrs`），SM-2 作为可选；**不自造算法**（ADR-004）
- 用户学习档案（模块 0.4）——当前学习语言 / 目标 / 水平 / 偏好，关联 `user` 表
- 牌组管理（1.1）+ 卡片管理（1.2，至少正反词卡）
- 复习会话（1.3）——极简界面，4 键 Again/Hard/Good/Easy（算法后台跑，参数隐藏）
- 学习队列（1.4）——新卡 / 到期卡 / 重学卡，每日上限

**验收线**（必须全绿）
```
建牌组 → 手动加几张卡 → 进入复习会话 → 4 键评分
  → FSRS 排期生效（写回 due/stability/difficulty 等状态）
  → 明天到期的卡，明天正确出现在队列里
```
即「**单人能真正用它背一轮词**」端到端可用。

**可后置到 v0.1.x**（不阻塞验收）
- 预置精选牌组（ADR-005 优先级高，但首版可只内置 1 套示例或暂缺）
- CSV / 文本导入（1.5ⓑ）、`.apkg` 完整解析（1.5ⓒ）、`.apkg` 导出（1.6）
- cloze / 释义卡等卡片类型（1.2 其余）
- SM-2 算法切换 UI（引擎先支持，设置项可后补）

**关键技术决策**（详细取舍待 `docs/modules/` 拆解时拍板）
- ⏳ 复习按钮默认 4 键还是简化 2 键（ADR 列为待定）
- ⏳ FSRS 卡片状态字段如何映射进 Drizzle schema（`ts-fsrs` 的 `Card` 结构 ↔ 表列）
- 语言相关字段（如所属学习语言）从一开始就**带上但不硬编码 English**（ADR-003）

## v0.2 · 语境化生成 ⭐

> 平台**杀手锏**，第二步就上。用最近学的词喂 AI 生成自然短文，把孤立单词织回语境。

**目标**：选「最近学/到期/某牌组/手选」的词 → AI 生成自然含这些词的短文 → 在文章学习器里读。

**交付物**
- 语境化复习文章（2B.1，**词驱动**）——选词集 → 生成短文
- 文章学习器（2B.3 雏形）——阅读视图（逐句语法标注 / TTS / 复述自测可渐进补）
- 生成历史库（2B.4 雏形）——存档生成结果
- **Inngest 异步管线**（架构决策，见下）
- 计时器适配器雏形（0.5）——阅读场景接入计时（为 v0.5 统计铺垫）

**架构决策（已拍板）：v0.2 直接上 Inngest 完整异步架构**
```
server action 发事件 → Inngest 后台调 chatCompletion 生成 → 前端轮询结果
```
不做「同步直连」过渡版。AI 生成可能 >30s，从一开始就规避 Vercel 60s 超时，避免后期重构。
> 生成状态需落库（pending/processing/ready/failed），前端据此轮询/展示 loading。

**验收线**
```
在某牌组/最近学的词里勾选一批词 → 触发生成
  → 后台异步产出一篇自然含这些词的短文 → 前端轮询到结果并展示
  → 该文章存入历史库可再次打开
```

**沿用约束**
- ADR-011：读语境文章**暂不**计为复习、不反哺 FSRS 排期（保持排期数据纯净）
- ⏳ 「最近学的词」取最近几天 / 多少个词——实现时定
- ⏳ 生词可控：先纯 prompt 约束，失控后再加词频表校验兜底

## v0.3 · 语法文章 + 记忆增强

> 复用 v0.2 的生成管线，补齐另一条生成线（语法驱动）和挂在单卡上的 AI 外援。

**目标**：语法点驱动生成背诵文章 + 一键助记/易混辨析。

**交付物**
- 语法背诵文章（2B.2，**语法点驱动**）——选语法点 → 富语法、生词可控的短文（复用 v0.2 Inngest 管线）
- 一键助记法（2A.1）——谐音 / 词根词缀 / 联想 / 故事，目标「终身难忘」
- 易混词辨析（2A.2）——拉出形近/义近词，**颜色高亮拼写差异 + 释义差异**，给区分记忆法
- 增强内容沉淀（2A.3）——助记/辨析固化为卡片字段，复习时复现（成为「记忆锚点」）

**验收线**
```
选一个语法点 → 生成富含该语法的短文（生词可控）
对一张难词卡 → 一键助记得到记忆法 / 一键辨析得到高亮对比 → 可固化进卡片字段
```

**依赖说明**
- 语法点清单来自**语言包**（0.3）——首发英语先内置一份；语言包接口此版可能首次成形（但仍只服务英语，不强行通用化）
- ⏳ 助记法默认偏好哪种风格（谐音/词根/联想）——实现时定

## v0.4 · 造卡回流 + 测验

> 有内容可沉淀/检验后再上。打通「回流闭环」与「测验闭环」，平台从「学习器」升级为「训练器」。

**目标**：AI 把文章/错题/助记沉淀为卡片回流模块 1 + 测验引擎检验产出。

**交付物**
- AI 造卡（2C.x）——从文章选中词/句、从测验错题、从助记辨析结果 → 卡片，落入指定牌组（**统一回流模块 1**）
- 测验引擎（3.1）——出题 → 作答 → 判分 → 错题沉淀，**题型可插拔**抽象层
- AI 题目生成（3.2）——由词/语法点/文章生成
- 基础题型（3.3）——词汇/语法选择改错/完形/翻译（四六级·雅思等标准化题型 🅿️ **预留**，本版不做）
- 错题本（3.4）——自动沉淀，一键转卡 → 走 2C 造卡通道
- 测验报告（3.5）——得分 / 薄弱点分析

**验收线**
```
回流闭环: 文章里选词 → 一键造卡 → 落入牌组 → 进入 v0.1 复习队列
测验闭环: 由一批词出题 → 作答判分 → 错题进错题本 → 一键转卡 → 回流复习
```

**关键决策**
- ADR-009/010：题型做成可插拔抽象；错题↔Anki 通过 2C 造卡通道实现（接受模块 3 与模块 1 强耦合）
- ⏳ 复述评分先纯自检，还是直接「语音复述 + AI 评分」——实现时定（此项也可能回填到 2B.3）

## v0.5 · 统计与成就

> 数据攒够才有意义。把前序所有行为数据化，制造真实成就感与分享欲（ADR：成就感驱动，非游戏化套路）。

**目标**：复习/学习时间/生成测验三类统计 + 成就分享。

**交付物**
- 复习统计（4.1）——复习日历热力图 / 到期预测 / 留存率 / 卡片成熟度分布（参考 Anki，用 `recharts`）
- **学习时间统计**（4.2）⭐——背单词/阅读/测验**独立计时 + 总和**，连续天数 streak；**计时器适配器完整化**（0.5，把 v0.2 雏形升级为统一抽象，接入全部三类场景）
- 生成与测验统计（4.3）——文章产量 / 测验趋势 / 薄弱语法点与词汇
- 成就与分享（4.4）——里程碑徽章 / 截图友好的成就卡

**验收线**
```
复习/阅读/测验各自计时并汇总为「平台有效学习总时长」
复习日历热力图 + streak 正确反映历史行为
达成某里程碑 → 解锁徽章 → 生成可截图分享的成就卡
```

**依赖说明**
- ADR-012：计时器适配器在此版从「v0.2 阅读雏形」升级为统一抽象，让「新增可计时场景」零成本接入
