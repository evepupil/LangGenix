# 产品愿景与方法论

## 一句话定位

**以 Anki（间隔重复）为底座、AI 生成为灵魂的一体化语言学习平台**——
把"学单词 → 复习 → 语境化 → 测验检验 → 统计反馈"做成一条咬合的闭环，
让学习者用最科学高效的方式真正"学会"一门语言，而不是停在"打卡"。

## 起点：一个被验证的学习方法

最初的洞察是一种朴素但极有效的方法：

> 让 AI 生成一篇文章——生词不多，但句式、语法特性密集——学习者反复读、反复背，
> 直到滚瓜烂熟、能随嘴说出，从而把语言"刻进肌肉记忆"。

它背后有扎实的二语习得理论支撑：

- **语块（chunking）**：大脑以高频搭配整块存储语言（如 `the more... the more...`），
  背熟整段 = 把语块连同真实语境刻进记忆，远胜背孤立语法规则。
- **可理解输入（comprehensible input, i+1）**：生词控制在少量，大脑才有余力吸收**结构**而非卡在查词。
- **自动化（automaticity）**：反复到"随嘴说出"，把语言从"需思考的知识"变成"不假思索的本能"——这就是流利度。

这一方法在平台中落地为 **模块 2B · 文章学习**，并通过 TTS 跟读、复述自测进一步强化（输入 + 输出）。

## 为什么过去没人把它做成产品

1. **内容生成成本**：这套方法的命脉是"生词可控 + 语法密集 + 长度精准 + 可标注"的文章。
   在 LLM 之前只能人工写，无法为每个用户/水平/话题规模化个性化。**LLM 让这一步第一次变得几乎免费。**
2. **商业模式冲突**：传统教育 App 靠"留存 / 打卡 / 课程包"赚钱，而最有效的方法往往**简单、免费、学会即走**，
   没有商业护城河，于是没人愿意把它做精（李阳"喊"而非做软件，同理）。
3. **有效但反人性**：纯背诵枯燥，留存数据难看，大厂 KPI（留存/时长/付费）会把产品推向"好玩"而非"高效"。

**LangGenix 的破局点**：自用优先、不伺候 KPI，把每个决策都压在"最科学高效"。
前两个历史障碍已被 LLM + TTS 打破，第三个障碍对"自用工具"不成立。

## 不止于"学习"，更要"检验"

> "我们平台不仅仅是学习、复习而已，还要有测验。是骡子是马拉出来溜溜。"

只输入不输出是半个产品。**模块 3 · 测验**把平台从"学习器"升级为"训练器"：
由词 / 语法点 / 文章生成题目，作答判分，错题沉淀回流为卡片。
标准化题型（四六级 / 雅思）作为预留扩展，引擎先做成题型可插拔的抽象。

## 北极星

- **效果**：用户在平台学的东西，能在真实场景（开口、考试）拉出来用。
- **闭环黏性**：靠"学 → 复习 → 语境化 → 测验 → 统计"两条闭环的咬合产生黏性，而非靠游戏化套路。
- **成就感驱动**：学习时间统计 + 连续天数 + 成就卡，制造真实的成就感与分享欲。

## 多语言战略

语言学习的核心闭环**语言无关**，仅少数维度**语言相关**（词频表、语法清单、TTS 语音、书写系统）。
因此采取 **单项目 + 语言包（Language Pack）配置驱动** 架构：**"加一门语言 = 加一份配置"**。
首发英语；抽象由"第二门语言"倒逼，避免只适配英语假设的伪通用接口。详见 [decisions.md](./decisions.md) ADR-003。
