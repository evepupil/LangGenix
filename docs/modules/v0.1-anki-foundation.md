## 验收线（Definition of Done）

```
建牌组 → 手动加几张正反词卡 → 进入复习会话 → 4 键评分
  → FSRS 排期写回卡片状态(due/stability/difficulty/state...)
  → 明天到期的卡，明天正确出现在队列里
  → 内置示例牌组开箱即可复习
```
即「**单人能真正用它背一轮词**」端到端可用。

## 决策定案

| # | 决策 | 结论 | 依据 |
| --- | --- | --- | --- |
| D1 | 复习按钮 | **4 键** Again/Hard/Good/Easy；2 键留后续设置项 | 标准 FSRS 体验、排期最准 |
| D2 | 卡片类型 | **只做正反词卡**（`basic`）；type enum 带上，cloze/释义卡留 v0.1.x | 验收线只要能背词，最快交付 |
| D3 | 主键 id | **`crypto.randomUUID()`** 调用处生成，schema 主键无默认值 | 跟随 credits/webhook 现有风格，不混入新风格 |
| D4 | 学习语言字段 | **`deck.learningLanguage`** 存语言码（`en`/`ja`…）；`profile.currentLanguage` 存当前在学 | ADR-003 不硬编码 English，多语言时牌组天然隔离 |
| D5 | 示例牌组 | **内置一小套** ~25 常用英词 seed，开箱可背 | 启动即可验收闭环；ADR-005 的精选牌组仍留 v0.1.x |
| D6 | SRS 抽象 | v0.1 **只实装 FSRS**，但包成算法适配器接口为 SM-2 预留 | ADR-004 双算法；ADR-003 抽象由第二个算法倒逼，先留接口不实装 |

## 一、数据模型（`src/db/schema.ts` 追加）

### 新增 enum（3 个）

```ts
cardTypeEnum   = pgEnum("card_type", ["basic"])                              // D2：仅 basic，未来加 cloze/definition
cardStateEnum  = pgEnum("card_state", ["new","learning","review","relearning"]) // 对齐 ts-fsrs State
srsAlgorithmEnum = pgEnum("srs_algorithm", ["fsrs","sm2"])                   // D6：默认 fsrs
```

> **学习语言不设 enum**（D4/ADR-003）：用 `text` 存语言码，避免把已知语言硬编码进类型，加语言无需改 schema。

### 表 1 · `user_learning_profile`（模块 0.4）

每用户一条，关联 `user`。存「当前在学哪门语言 + 学习偏好」。

| 字段 | 类型 | 说明 |
| --- | --- | --- |
| `id` | text PK | `crypto.randomUUID()` |
| `userId` | text FK→user, unique, cascade | 一人一份 |
| `currentLanguage` | text, default `'en'` | 当前学习语言码（D4，**非** next-intl locale） |
| `algorithm` | srsAlgorithmEnum, default `'fsrs'` | 排期算法偏好（D6） |
| `dailyNewLimit` | integer, default 20 | 每日新卡上限（1.4） |
| `dailyReviewLimit` | integer, default 200 | 每日复习上限（1.4） |
| `reviewButtonMode` | text, default `'four'` | `'four'`/`'two'`（D1，先只用 four） |
| `createdAt`/`updatedAt` | timestamp | |

### 表 2 · `deck`（模块 1.1）

| 字段 | 类型 | 说明 |
| --- | --- | --- |
| `id` | text PK | randomUUID |
| `userId` | text FK→user, cascade | 牌组归属 |
| `name` | text notNull | 牌组名 |
| `description` | text nullable | 描述 |
| `learningLanguage` | text notNull, default `'en'` | D4：所属学习语言码 |
| `createdAt`/`updatedAt` | timestamp | |

### 表 3 · `card`（模块 0.1 + 1.2）— 内嵌 FSRS 状态

内容字段 + FSRS 调度状态**同表内嵌**（FSRS `Card` 是值对象，无需独立表）。FSRS 字段名/类型按 `ts-fsrs` 的 `Card` 接口核实结果落列。

| 字段 | 类型 | 说明 |
| --- | --- | --- |
| `id` | text PK | randomUUID |
| `deckId` | text FK→deck, cascade | 所属牌组 |
| `userId` | text FK→user, cascade | 冗余存便于按用户查队列 |
| `type` | cardTypeEnum, default `'basic'` | D2 |
| `front` | text notNull | 正面（词） |
| `back` | text notNull | 背面（释义） |
| **— FSRS 状态 —** | | 见 `ts-fsrs` `Card` 接口 |
| `due` | timestamp notNull | 下次到期时间（队列核心） |
| `stability` | real notNull, default 0 | 记忆稳定性 |
| `difficulty` | real notNull, default 0 | 难度 |
| `scheduledDays` | integer notNull, default 0 | 排期间隔天数 |
| `reps` | integer notNull, default 0 | 复习次数 |
| `lapses` | integer notNull, default 0 | 遗忘次数 |
| `state` | cardStateEnum, default `'new'` | 卡片状态 |
| `learningSteps` | integer notNull, default 0 | 学习阶段步数（ts-fsrs v5+） |
| `lastReview` | timestamp nullable | 上次复习（ts-fsrs 唯一可空字段） |
| `createdAt`/`updatedAt` | timestamp | |

每次评分一条。支撑：复习撤销、v0.5 统计（热力图/留存率）。映射 ts-fsrs `ReviewLog`。

| 字段 | 类型 | 说明 |
| --- | --- | --- |

| `rating` | integer notNull | 1=Again 2=Hard 3=Good 4=Easy（ts-fsrs Rating） |
| `state` | cardStateEnum notNull | 复习**前**状态 |

| `stability`/`difficulty` | real notNull | 复习后快照 |
| `scheduledDays` | integer notNull | |

## 二、SRS 引擎（`src/features/srs/`）

把 `ts-fsrs` 封成**算法适配器**，对上层暴露稳定接口；v0.1 只实装 FSRS（D6）。

```
src/features/srs/
├── types.ts          # CardSchedulingState（与 card 表 FSRS 列同构）、Rating、SchedulerResult
├── scheduler.ts      # SrsScheduler 接口：schedule(state, rating, now) → { next, log }
├── fsrs-adapter.ts   # 用 ts-fsrs 实现 SrsScheduler；DB 行 ↔ ts-fsrs Card 互转
├── index.ts          # getScheduler(algorithm) 工厂，默认 fsrs
└── __不在此__         # 测试在 src/test/srs/（项目约定测试不 colocate）
```

**核心纯函数**（可单测）：
- `toFsrsCard(row): Card` / `fromFsrsCard(card): CardSchedulingState` —— DB ↔ ts-fsrs 字段互转（含 `state` 字符串 enum ↔ ts-fsrs 数字 State）
- `schedule(state, rating, now)` —— 调 `fsrs().next(card, now, rating)`，返回更新后状态 + log

> ts-fsrs 的 `State` 是数字枚举（New=0/Learning=1/Review=2/Relearning=3），DB 存字符串 enum——互转映射在 adapter 内集中处理，唯一耦合点。

## 三、Server Actions（`src/features/deck/`、`src/features/card/`、`src/features/review/`）

全部用 `protectedAction`（提供 `ctx.userId`），命名遵 `feature.<name>` 元数据约定。id 用 `crypto.randomUUID()`（D3）。

### 牌组（`features/deck/actions/`）
- `createDeck({ name, description?, learningLanguage })`
- `updateDeck({ id, ... })` / `deleteDeck({ id })`（级联删卡）
- 查询：列表在 RSC 直接用 Drizzle 读（项目约定 RSC 直查），不走 action

### 卡片（`features/card/actions/`）
- `createCard({ deckId, front, back })` —— 新卡初始 FSRS 状态 = `createEmptyCard()` 映射（state=new, due=now）
- `updateCard` / `deleteCard`

### 复习（`features/review/actions/`）— 闭环核心
- `getReviewQueue({ deckId })` —— **队列计算**（纯逻辑，单测覆盖）：
  - 到期卡：`due <= now` 且 state≠new
  - 新卡：state=new，受 `dailyNewLimit` 限
  - 重学卡：state=relearning 且到期
  - 受 `dailyReviewLimit` 限；排序：重学 > 到期 > 新卡
- `reviewCard({ cardId, rating })` —— 取卡 → `schedule()` → 事务内 `UPDATE card` + `INSERT review_log`
- `undoReview({ cardId })`（可选，v0.1.x；review_log 已为它留好数据）

## 四、UI（`src/app/[locale]/(dashboard)/` + 各 feature 的 components）

纯展示层，**不写单测**（项目测试规范）。i18n 导航从 `@/i18n/routing` 导入。

| 页面 | 路由 | 内容 |
| --- | --- | --- |
| 牌组列表 | `(dashboard)/decks` | 牌组卡片网格 + 各组到期/新卡数 + 新建入口 |
| 牌组详情 | `(dashboard)/decks/[deckId]` | 卡片列表 + 增删改卡 + 「开始复习」入口 |
| 复习会话 | `(dashboard)/decks/[deckId]/review` | 极简：显示 front → 翻面显 back → **4 键**评分 → 下一张；队列空显示完成 |

- 复习会话 `'use client'`（交互），4 键按钮可显示各档「下次间隔预览」（ts-fsrs `repeat()` 给四档预览，锦上添花，可后置）
- 复杂参数（stability/difficulty）**默认不展示**（ADR-004 藏复杂度）

## 五、示例牌组 seed（D5）

`src/features/deck/seed/sample-deck.ts` —— ~25 个常用英语词（front=词，back=释义），提供 `seedSampleDeck(userId)`：用户首次进 `/decks` 且无牌组时自动建一套 `learningLanguage='en'` 的示例牌组。轻量、纯数据。

> 这是「示例」非「精选」——ADR-005 的高质量精选牌组（考研/四六级核心词）仍按 roadmap 留 v0.1.x。

## 七、实现顺序（建议提交粒度）

4. **review actions（队列 + reviewCard）+ 单测**（1 commit）
