# v0.2 · 语境化生成 — 详细实现拆解

> 本文是 v0.2 的**施工图**，由 [`roadmap.md`](../roadmap.md) 的 v0.2 里程碑展开。
> 已含拍板决策（见[决策定案](#决策定案)）。实现严格按本文，完成后跑门禁 + commit。
> 涉及模块：2B.1 词驱动语境文章 · 2B.3 文章学习器(阅读视图) · 2B.4 生成历史库 · 0.5 计时雏形 · Inngest 异步管线。

---

## 验收线（Definition of Done）

```
在某牌组「用最近学的词」(或手选覆盖) → 触发生成
  → Inngest 后台用 DeepSeek 产出一篇自然含这些词的短文
  → 前端通过 Inngest 实时流(失败时降级轮询 article 表)感知完成
  → 文章进入历史库，可再次打开阅读
```
即「**选词 → AI 生成语境短文 → 阅读**」闭环可用。读文章**不计为复习**（ADR-011）。

## 一、数据模型（`src/db/schema.ts` 追加）

### 新增 enum
```ts
articleStatusEnum = pgEnum("article_status", ["pending", "processing", "ready", "failed"])
```

### 表 · `article`（生成历史库 + 状态轮询载体）

| 字段 | 类型 | 说明 |
| --- | --- | --- |
| `id` | text PK | randomUUID |
| `userId` | text FK→user, cascade | 归属 |
| `deckId` | text FK→deck, cascade nullable | 来源牌组（可选） |
| `title` | text nullable | AI 生成或派生，ready 后填 |
| `sourceWords` | jsonb notNull | 喂给 AI 的词集（`{front, back}[]` 快照） |
| `content` | text nullable | 生成正文，ready 后填 |
| `status` | articleStatusEnum, default `pending` | 生成状态（轮询/实时流依据） |
| `model` | text nullable | 实际用的 AI 模型名（供回溯） |
| `errorMessage` | text nullable | failed 时的错误信息 |
| `createdAt`/`updatedAt` | timestamp | |

> `sourceWords` 存快照而非引用 card id——卡片可能被删/改，文章应保留生成时的词义语境。

## 二、Inngest 异步管线

### 事件（`src/inngest/client.ts` 加类型）
```ts
"article/generate": { data: { articleId: string } }
```

### 函数（`src/inngest/functions.ts` 加）
```
articleGenerate = inngest.createFunction(
  { id: "article-generate", retries: 2 },
  { event: "article/generate" },
  async ({ event, step }) => {
    step.run("load-article", 取 article + sourceWords)
    step.run("generate", 调 chatCompletion 生成，失败抛错→status=failed)
    step.run("persist", 写回 content/title/model, status=ready)
  }
)
```
注册进 `functions` 数组。

### 触发（`features/article/actions/`）
- `generateArticleAction({ deckId, words })` —— 校验归属 → 建 `article`(pending) → `inngest.send("article/generate", {articleId})` → 返回 articleId 供前端跟踪

## 三、生成器（`src/features/article/generator.ts`）

纯函数：构造 prompt → 调 `chatCompletion`。

- **prompt 策略**（D6 纯约束）：
  - system：你是语言学习内容生成器，用给定词写一篇自然短文
  - 约束：① 必须包含所有给定词 ② 生词量可控（除给定词外只用常见词）③ 长度精准（~150-250 词）④ 适合该学习语言水平 ⑤ 附一个标题
  - user：词集（front=词，back=释义供 AI 理解语境）+ 学习语言 + 水平
- **输出解析**：让 AI 返回 `{title, content}` JSON（用 `jsonMode: true`），解析失败则整体当 content、title 用首行。

---

## 四、查询（`src/features/article/queries.ts`）

- `getArticle(id, userId)` —— 取单篇（校验归属）
- `getUserArticles(userId)` —— 历史库列表（按 createdAt 倒序）
- `getRecentLearnedWords(deckId, userId, limit=10)` —— **D3 自动取最近**：该牌组下 `lastReview != null` 的卡，按 `lastReview` 倒序取 N 个（返回 `{id, front, back}`）。新卡（`state='new'`）不算"已学"。

## 五、UI

### 路由（RSC + client 混合）
| 页面 | 路由 | 内容 |
| --- | --- | --- |
| 选词 + 触发生成 | 牌组详情页加入口 → **`GenerateArticleDialog`**（client） | 自动预填最近 10 词，可勾选调整 → 提交触发 action |
| 生成中 + 阅读 | `(dashboard)/articles/[articleId]/page.tsx` | 状态 pending/processing 显示进度（实时流/轮询），ready 显示阅读视图 + 阅读计时 |
| 历史库 | `(dashboard)/articles/page.tsx` | 文章列表，点击进阅读 |

### 组件（`features/article/components/`）
- `GenerateArticleDialog` —— 选词 + 触发
- `ArticleReader` —— 阅读视图（排版正文）+ 阅读计时（前端累计，D7）
- `ArticleStatusPoller` —— **实时流优先 + 降级轮询**：先尝试 Inngest stream；连不上或超时则每 1.5s 查 `getArticle`，status 到 ready/failed 停

### 导航
`config/nav.ts` 加「文章」入口 → `/dashboard/articles`

## 十、环境前置（需用户处理）

- ⛔ **DeepSeek API key**：在 `.env.local` 填 `DEEPSEEK_API_KEY=sk-...`（真实值），并确认 `AI_PROVIDER=deepseek`。无 key 则生成会失败（article→failed）。
- ⚠️ **Inngest dev server**（实时流需要）：本地额外跑 `npx inngest dev`。不跑则实时流不可用，前端自动降级轮询（功能仍可用，只是感知完成慢 ~1.5s）。
