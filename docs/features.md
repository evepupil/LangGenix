# 功能架构总纲（Feature Map）

> 本文是 LangGenix 的功能纲领（模块树 + 子功能 + 分层 + 闭环）。
> 所有拍板取舍见 [`decisions.md`](./decisions.md)。逐模块的详细需求待开发阶段再补。

## 核心认知：四大功能是一条流水线，不是四个孤岛

平台围绕 **「单词 / 卡片」** 这一核心数据，在不同阶段做不同的事：

```
        ┌─────────── Anki 卡片/牌组（核心数据底座）───────────┐
        │                                                      │
  学单词            复习排期             语境化               度量
  (导入/预置)   →  (SRS 间隔重复)   →  (AI 用最近学的词    →  (统计)
                                        生成语境文章)
                                              │
                                        (主线之外)
                              语法文章 (按语法点生成, 背诵+理解)
```

**地基是 Anki 数据模型**：语境化文章、记忆增强、测验、统计全都长在它上面。

## 模块树

### 模块 0 · 基础底座（地基，语言无关）
- **0.1 卡片 / 牌组数据模型** — 平台核心数据，其余模块皆依赖
- **0.2 SRS 排期引擎** — FSRS / SM-2 **双算法可选**，默认 FSRS（`ts-fsrs`），复杂度默认隐藏
- **0.3 语言包（Language Pack）** — 每语言一份：词频分级表 / 语法点清单 / TTS 语音 / prompt 模板
- **0.4 用户与学习档案** — 当前学习语言、目标、水平档位、偏好
- **0.5 计时器适配器** ⭐ — 统一计时抽象，被刷卡 / 阅读 / 测验接入，各场景独立计时 + 汇总

### 模块 1 · Anki 学习（核心学习形态之一：管「词」）
- 1.1 牌组管理（创建 / 编辑 / 删除 / 分类）
- 1.2 卡片管理（正反词卡 / 填空 cloze / 释义卡）
- 1.3 复习会话（极简界面，4 键 Again/Hard/Good/Easy 或简化 2 键，算法后台跑）
- 1.4 学习队列（新卡 / 到期卡 / 重学卡，每日上限）
- 1.5 导入（ⓐ 预置精选牌组 ⓑ CSV / 文本 ⓒ 完整 `.apkg`）
- 1.6 导出（`.apkg` 回流桌面 Anki）

### 模块 2A · AI 记忆增强（Anki 做不到的部分）⭐
> 挂在单个卡片/单词上的"即时 AI 外援"，破解"记不住 / 记混"。
- 2A.1 一键助记法（谐音 / 词根词缀 / 联想画面 / 故事，目标"终身难忘"）
- 2A.2 易混词辨析（拉出形近/义近词，**颜色高亮拼写差异 + 释义差异**，给区分记忆法）
- 2A.3 增强内容沉淀（助记 / 辨析固化为卡片字段，成为永久"记忆锚点"，复习时复现）

### 模块 2B · 文章学习（核心学习形态之二：管「语境 + 语法」）
> 最初的出发点，与模块 1 并列的另一种学习方式。
- 2B.1 语境化复习文章（**词驱动**）— 选最近学/到期/某牌组/手选的词 → 生成自然含这些词的短文
- 2B.2 语法背诵文章（**语法点驱动**）— 选语法点 → 富语法、生词可控的短文，目标"背诵 + 理解"
- 2B.3 文章学习器（共用）— 阅读视图 + 逐句语法标注 + TTS 跟读 + 复述/产出自测
- 2B.4 生成历史库 — 存档 / 收藏 / 再练

### 模块 2C · AI 造卡（沉淀通道，完全新做）
- 2C.1 从文章造卡（选中词/句 → 卡片）
- 2C.2 从测验错题造卡
- 2C.3 从助记 / 辨析结果造卡
- 2C.4 批量 / 单卡造卡，落入指定牌组（统一回流模块 1）

### 模块 3 · 测验 / 测试（输出层，预留标准化题型扩展）
- 3.1 测验引擎（出题 → 作答 → 判分 → 错题沉淀，**题型可插拔**抽象层）
- 3.2 AI 题目生成（由词 / 语法点 / 文章生成）
- 3.3 题型库（词汇 / 语法选择改错 / 完形 / 翻译；**预留四六级·雅思**等标准化题型）
- 3.4 错题本（自动沉淀，一键转卡 → 模块 2C）
- 3.5 测验报告（得分 / 薄弱点分析）

### 模块 4 · 统计与成就（驱动坚持与分享欲）
- 4.1 复习统计（复习日历热力图 / 到期预测 / 留存率 / 卡片成熟度分布，参考 Anki）
- 4.2 **学习时间统计** ⭐（背单词 / 阅读 / 测验**独立计时 + 总和**，连续天数 streak；源自 0.5）
- 4.3 生成与测验统计（文章产量 / 测验趋势 / 薄弱语法点与词汇）
- 4.4 成就与分享（里程碑徽章 / 截图友好的成就卡）

### 模块 5 · 系统与设置（横切）
- 5.1 语言切换（驱动语言包 0.3）
- 5.2 学习偏好（算法 / 每日上限 / 复习按钮模式 / 难度档）
- 5.3 AI / TTS 配置（模型与语音选择）
- 5.4 数据管理（备份 / 导出 / 清理）

## 分层图

```
┌───────────────────────────────────────────────────────────────────┐
│  模块4 统计成就      模块3 测验/测试        模块5 系统设置            │  展示/输出/横切
├───────────────────────────────────────────────────────────────────┤
│  模块2B 文章学习  │ 模块2A AI记忆增强 │ 模块2C AI造卡(回流→模块1)    │  AI 灵魂层
│  (语境/语法)      │ (助记/易混辨析)   │                              │
├───────────────────────────────────────────────────────────────────┤
│  模块1 Anki学习 (牌组/卡片/复习/导入导出)  ← 两大核心学习形态之一    │  数据操作层
├───────────────────────────────────────────────────────────────────┤
│  模块0 底座 (卡片模型·SRS引擎·语言包·用户档案· [计时器适配器]横切)   │  地基
└───────────────────────────────────────────────────────────────────┘
```

## 两条核心闭环（产品黏性来源）

```
两大学习形态:  [模块1 刷卡 = 管词]   ‖   [模块2B 文章 = 管语境/语法]

主闭环:  学词 → 复习 → (2A 增强记忆 / 2B 语境化) → 测验检验 → 统计反馈

回流闭环:  文章新词 · 测验错题 · 助记辨析 ──(模块2C 造卡)──► 回到模块1 复习

计时横切:  模块0.5 计时器适配器 ──接入──► 刷卡 / 阅读 / 测验 ──► 模块4.2 汇总
```

## 落地顺序（每步都能立刻自用）

| 版本 | 内容 | 依赖逻辑 |
| --- | --- | --- |
| **v0.1 Anki 底座** | 卡片/牌组模型 + SRS 排期 + 极简复习界面 + 一套预置牌组 | 先有数据底座 |
| **v0.2 语境化生成** | 用"最近学的词"喂 AI 生成短文（功能 2B.1）+ 文章学习器 | 杀手锏，第二步上 |
| **v0.3 语法文章 + 记忆增强** | 语法点驱动文章（2B.2）+ 一键助记/易混辨析（2A） | 复用 v0.2 管线 |
| **v0.4 造卡回流 + 测验** | AI 造卡（2C）+ 测验引擎与基础题型（3） | 有内容可沉淀/检验后 |
| **v0.5 统计成就** | 复习统计 + 学习时间 + 成就（4） | 数据攒够才有意义 |

> 顺序逻辑：**先有数据底座（模块1）→ 才有东西可生成（2B/2A）→ 才有东西可沉淀检验（2C/3）→
> 数据攒够才做统计（4）**，与模块依赖关系完全一致。
